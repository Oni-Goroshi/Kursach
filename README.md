# 🔐 Журнал паролей

Локальный менеджер паролей с шифрованием **RSA-2048 / OAEP** на C++ и Qt 6.

## О проекте

Приложение позволяет хранить пароли от любых сервисов на вашем компьютере в зашифрованном виде. Для шифрования используется промышленный стандарт **RSA-2048 с OAEP-паддингом** через библиотеку **OpenSSL** — тот же алгоритм, что защищает банковские системы и HTTPS-соединения.

Все пароли хранятся в файле `passwords.dat` в формате **Base64**. Без знания мастер-пароля расшифровать их вычислительно невозможно.

## Функциональность

| Функция | Описание |
|---|---|
| 🔑 **Мастер-пароль** | При запуске вводится мастер-пароль — из него через SHA-256 детерминированно генерируется пара ключей RSA-2048 |
| ➕ **Добавление записи** | Сервис, логин, пароль, заметка |
| 👁️ **Просмотр пароля** | RSA-OAEP дешифрование и показ пароля выбранной записи |

| 💾 **Сохранение** | Запись зашифрованного журнала в `passwords.dat` |
| 📂 **Загрузка** | Автозагрузка при запуске; повторная загрузка по кнопке |

## Алгоритм шифрования

### RSA-2048 + OAEP + SHA-256

**Генерация ключей** (при каждом запуске):
```
SHA-256(мастер-пароль) → 32-байтовый seed → RAND_seed (OpenSSL CSPRNG)
                                                    ↓
                              EVP_PKEY_keygen → пара ключей RSA-2048
                              (p, q — случайные 1024-битные простые числа)
```

**Шифрование пароля:**
```
Пароль (UTF-8) → RSA-OAEP-SHA256 → 256 байт шифртекста → Base64 (344 символа)
```

**Дешифрование:**
```
Base64 → 256 байт шифртекста → RSA-OAEP-SHA256 → исходный пароль (UTF-8)
```

**Почему OAEP?** Голый RSA уязвим к атакам на основе выбранного шифртекста (chosen-ciphertext attack). OAEP (PKCS#1 v2.1) добавляет случайный компонент и обеспечивает семантическую стойкость — стандарт для всех современных реализаций RSA.

**Детерминированность:** SHA-256 от одинакового мастер-пароля всегда даёт одинаковый seed, OpenSSL генерирует те же ключи — сохранённые пароли расшифровываются при каждом запуске.

## Сборка и запуск

### Требования

- **Qt 6.x** — [скачать установщик](https://www.qt.io/download-qt-installer)  
  При установке выбрать: `Qt 6.x` → `MinGW 64-bit` (Windows) или `GCC 64-bit` (Linux)
- **OpenSSL 3.x** — установка зависит от ОС (см. ниже)

### Установка OpenSSL

**Windows:**  
Скачать с [slproweb.com/products/Win32OpenSSL.html](https://slproweb.com/products/Win32OpenSSL.html) — выбрать **Win64 OpenSSL v3.x.x** (не Light).  
Установщик размещает библиотеки в `D:\OpenSSL-Win64\lib\VC\x64\MD\` — путь уже прописан в `.pro` файле.  
Если установили OpenSSL в другую папку — измените `OPENSSL_DIR` в `PasswordJournal.pro`.

**Linux (Ubuntu / Debian):**
```bash
sudo apt install libssl-dev
```

**macOS:**
```bash
brew install openssl
```
На Apple Silicon (M1/M2/M3) раскомментируйте строку с `/opt/homebrew` в `PasswordJournal.pro`.

### Сборка через Qt Creator

1. Откройте **Qt Creator**
2. `Файл` → `Открыть файл или проект` → выберите `PasswordJournal.pro`
3. Выберите комплект **Desktop Qt 6.x MinGW 64-bit** → `Настроить проект`
4. Нажмите **▶** (Ctrl+R)

### Сборка через командную строку

```bash
git clone https://github.com/Oni-Goroshi/Kursach.git
cd Kursach

qmake PasswordJournal.pro
make            # Linux / macOS
mingw32-make    # Windows

./password_journal      # Linux / macOS
password_journal.exe    # Windows
```

### Ключевые классы

**`RSACipher`** — обёртка над OpenSSL EVP API:
- `generateKeys(masterPassword)` — SHA-256 → RAND_seed → RSA-2048 keygen
- `encrypt(text, keys)` — RSA-OAEP-SHA256 → Base64
- `decrypt(cipherB64, keys)` — Base64 → RSA-OAEP-SHA256 → UTF-8

**`MainWindow`** — главное окно Qt, реализует весь UI и логику работы с файлом.

**`PasswordEntry`** — структура одной записи: `service`, `login`, `password` (Base64), `note`.

## Формат файла passwords.dat

```
# Password Journal — RSA-2048/OAEP (OpenSSL)
# Saved: 2026-05-17 21:00:00
Google|ivan@gmail.com|JKm3Rf+a9Bx...AAAA==|Основная почта
VK|ivan2005|Tz8pQw+k1Cx...BBBB==|
GitHub|t.berandze|Xm4nRs+b2Dy...CCCC==|Рабочий аккаунт
```

Разделитель полей — `|`. Base64 его не содержит, поэтому разбиение всегда корректно. Каждая Base64-строка — 344 символа (256 байт RSA-шифртекста).

## Как пользоваться

1. **Запустите** программу → введите мастер-пароль (нигде не сохраняется)
2. **Добавьте запись:** `➕ Добавить` → введите сервис, логин, пароль, заметку
3. **Сохраните:** `💾 Сохранить` → данные запишутся в `passwords.dat`
4. **При следующем запуске** введите тот же мастер-пароль — данные загрузятся автоматически

> ⚠️ Мастер-пароль не хранится нигде. Если его забыть — расшифровать данные невозможно.
